1.Setting Up Bootstrap 5

Exercise 1.1 Basic HTML page linked to Bootstrap 5 via CDN

The fastest way to start: pull the CSS and JS straight from a CDN, no install required. This very page is built exactly this way.

This alert, the buttons below, and every component on this page are rendered using only the CDN links shown in the code.
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My Bootstrap Page</title>
  <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css" rel="stylesheet">
</head>
<body>
  <div class="container py-5">
    <h1>Hello, Bootstrap 5!</h1>
    <button class="btn btn-primary">Click me</button>
  </div>

  <!-- JS bundle goes just before </body> -->
  <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js"></script>
</body>
</html>
Exercise 1.2 Project set up with npm or downloaded files

For real projects you usually don't rely on a CDN. Two common routes:

Option A — npm

npm init -y
npm install bootstrap @popperjs/core

Option B — downloaded files

Download the compiled CSS/JS from getbootstrap.com, unzip it next to your HTML file, then reference the local paths instead of the CDN:

<link href="bootstrap-5.3.3-dist/css/bootstrap.min.css" rel="stylesheet">
...
<script src="bootstrap-5.3.3-dist/js/bootstrap.bundle.min.js"></script>
Use local/npm files when you need offline builds, version pinning, or Sass customization (see Section 19). Use the CDN for quick prototypes — like this page.

2.Bootstrap Structure and Files

Exercise 2.1 Structure of the downloaded Bootstrap directory

A downloaded Bootstrap 5 dist folder looks like this:

bootstrap-5.3.3-dist/
├── css/
│   ├── bootstrap.css            # full, unminified CSS
│   ├── bootstrap.min.css        # minified, production-ready CSS
│   └── bootstrap-grid.css       # grid system only, no components
├── js/
│   ├── bootstrap.js             # plugins, needs Popper separately
│   ├── bootstrap.bundle.js      # plugins + Popper bundled in
│   └── bootstrap.bundle.min.js  # minified version, used most often
└── icons/                       # only present if Bootstrap Icons added separately
    ├── bootstrap-icons.css
    └── fonts/
FolderPurpose
css/Stylesheets that provide the grid, typography, and component styling.
js/JavaScript behaviour for interactive components (modals, dropdowns, carousels, tooltips...).
icons/Optional icon font/SVG set (Bootstrap Icons), kept as a separate package from core Bootstrap.
Exercise 2.2 Include Bootstrap's JS plugins via bootstrap.bundle.min.js

bootstrap.bundle.min.js bundles Popper, so dropdowns, tooltips, and popovers work with one script tag.

  • Profile
  • Settings
  • Logout
<!-- right before </body> -->
<script src="js/bootstrap.bundle.min.js"></script>

<div class="dropdown">
  <button class="btn btn-secondary dropdown-toggle" data-bs-toggle="dropdown">
    Open dropdown
  </button>
  <ul class="dropdown-menu">
    <li><a class="dropdown-item" href="#">Profile</a></li>
  </ul>
</div>

19.Customization with Sass

Exercise 19.1 Set up a Bootstrap 5 project with Sass via npm
# 1. init project & install deps
npm init -y
npm install bootstrap sass --save-dev

# 2. project structure
my-project/
├── scss/
│   └── custom.scss
├── index.html
└── package.json

# 3. add a build script to package.json
"scripts": {
  "build:css": "sass scss/custom.scss css/custom.css"
}

# 4. compile
npm run build:css
<!-- index.html -->
<link href="css/custom.css" rel="stylesheet">
Exercise 19.2 Customize primary color and border radius via _variables.scss

Override Bootstrap's Sass variables before importing Bootstrap itself, so your values replace the defaults during compilation.

// scss/custom.scss

// 1. Your overrides FIRST
$primary: #ff6b35;
$border-radius: 1rem;
$border-radius-lg: 1.25rem;
$border-radius-sm: .5rem;

// 2. Then pull in Bootstrap's source Sass
@import "../node_modules/bootstrap/scss/bootstrap";
npm run build:css   # recompiles css/custom.css with the new
                     # primary color and rounder corners applied
                     # to every component (buttons, cards, alerts...)
This page itself uses the compiled defaults from the CDN, so it can't show the recompiled output live — but every btn-primary, card, and rounded element across the site would pick up the new orange/round-corner theme automatically once custom.css replaces the CDN link.
